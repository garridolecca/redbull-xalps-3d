  • Source: 37.5M GPS fixes from 35 athletes across 4 sensor types (Naviter, Flarm, OGN, Satellite)
  • Processing: Parsed pipe/semicolon-delimited PubSub messages, deduplicated via 60-second time buckets with sensor priority ranking, extracted thermal climb events (>1.0 m/s) and wind observations
  • Output: 439K track points, 80K thermal events, 62K wind observations covering 17 race days

Visualization Layers

  • 3D Track Lines — Each athlete's full route rendered as a glowing polyline at actual GPS altitude above the Alpine terrain. Dual-layer glow effect (wide translucent + narrow bright core)
  • Animated Athlete Markers — Colored dots that move along each track in real-time, with smooth interpolation between data points. Shows current altitude in the sidebar
  • Thermal Columns — 3D cylinders at locations where athletes detected rising air (vertical speed > 1 m/s). Height and width scale with climb rate. Color: gold (1 m/s) → orange → red → crimson (4+ m/s). Time-filtered to a ±1 hour window
  • Wind Observations — Colored dots showing windspeed measured by Naviter instruments. Intensity maps to speed. Toggle via Layers panel
  • Dynamic Lighting — Sun position updates automatically with the time slider, creating realistic day/night transitions across the Alps

Technical Details

  • Rendering: ArcGIS Maps SDK for JavaScript 4.30, SceneView with world elevation, absolute-height positioning
  • Performance: Binary search for time-filtered layers, throttled updates, capped at 4000 thermal / 3000 wind graphics per frame
  • Data Schema: 18 fields per GPS fix including lat, lon, altitude (GPS + barometric), groundspeed, vertical speed, windspeed, wind direction, activity mode (Flying/Hiking/Stationary), rest & nightpass status, GPS quality (SNR 0-50)
